# Endpoints usuario

Módulo implicado
Descripción

Facturación y Autogestión (F&A)

Se busca la información básica de un usuario a partir del token suministrado por app web/mobile. Esto nos permitirá poder "pintar" la aplicación frontend. :from puede tomar el valor de web o mobile y :token el token de la app cliente.

GET https://notflix-fya-backend.herokuapp.com/api/users/:from/:token 

sequenceDiagram
    F&A->>BD: GET user data by TOKEN
    alt Correct GET
        BD->>F&A: {code: 200, data: {...}}
    else Bad GET
         BD->>F&A: {code: 500, data: "ERROR-001", message: "Error"}
    end

Descripción

Facturación y Autogestión (F&A)

Se realiza el registro a un nuevo paquete.

POST https://notflix-fya-backend.herokuapp.com/api/subscriptions/internal 

// Example body request structure for new subscription registration

let payload = {
    "userId": 1,
    "email":"string", //email del usuario
    "name": "string",
    "subscriptionId": 1,
    "packageId": 4,
    "cost": 300
}
sequenceDiagram
    F&A->>BD: POST by :userId
    alt Correct POST
        BD->>F&A: {code: 201, data: {...}}
    else Bad POST
         BD->>F&A: {code: 404, data: "Cannot create invoice because userId doesn't exist"}
    end

Facturación y Autogestión (F&A)

Se realiza el cambio de estado de una factura. Este estado es utilizado para verificar que suscripción será eliminada en el futuro.

PATCH https://notflix-fya-backend.herokuapp.com/api/subscriptions/:userId/:subscriptionId

    else Bad PATCH

    end
